# 挖财收支记录 JSON 格式规范

## 图片解析提示词

解析图片中的收支记录为规范JSON，仅以JSON返回，不要添加多余的描述。收付账户统一为：<收付账户>，类型规则：还款->收入，转账->支出。

> 注意：使用时需将 `<收付账户>` 替换为实际账户名称（如"招商银行储蓄卡 1234"、"现金"、"微信钱包"等），根据图片实际内容判断。

## 字段说明

| 字段 | 说明 | 示例值 |
|------|------|--------|
| 日期时间 | 格式 `YYYY-MM-DD HH:MM:SS` | `2024-10-22 10:44:00` |
| 类型 | 支出 / 收入 / 转账 / 借贷 | `支出` |
| 类别 | 一级或二级类别，用 `/` 分隔 | `餐饮/早餐` |
| 金额 | 浮点数，正值 | `1.0` |
| 币种 | 货币名称 | `人民币` |
| 收付款人 | 对方姓名，借贷还款时填写 | `自己：-100.00，张三：+100.00` |
| 收付账户 | 付款账户；转账时格式为 `账户A：-金额, 账户B：+金额` | `现金` |
| 参与人 | AA制分摊，格式 `姓名：金额` | `自己：0.50, 张三：0.50` |
| 标签 | 逗号分隔 | `冗余,日常` |
| 商家 | 商家名称 | `超市` |
| 属性 | 如"已报销" | `已报销` |
| 备注 | 自由文本 | `` |

## 类型映射规则

- **还款** → 类型改为 `收入`
- **转账** → 类型改为 `支出`（如需区分方向则在收付账户字段体现）
- 借贷类：类别为"借入"或"借出"，类型填 `借贷`

## CSV 列顺序

导出 CSV 时必须保持以下列顺序（挖财标准导入格式）：

```
日期时间,类型,类别,金额,币种,收付款人,收付账户,参与人,标签,商家,属性,备注
```
